Collection

(the root interface of collection hierarchy)

public interface Collection<E>

extends Iterable<E>

E are the type of elements in this collection

The JDK does not provide any direct implementation of this interface.
Rather, it provides group of classes (i.e. ArrayList, Hashset)
and subinterfaces (i.e. List, Queue) that make up the collections framework.
Depending on the task in hand, some collections are better for
manipulating a group of elements than others.

Advantages

  • Consistent API : The API has a basic set of interfaces like Collection, Set, List. All classes (ArrayList, LinkedList, Vector, etc)
    • that implement these interfaces have some common set of methods.

  • Reduces programming effort:
    • A programmer doesn’t have to worry about the design of Collection, and he can focus on its best use in his program.

  • Increases program speed and quality:
    • Increases performance by providing high-performance implementations of useful data structures and algorithms.
